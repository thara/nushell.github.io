# Nu installieren

Der aktuell beste Weg Nu zu bekommen ist, es von crates.io (opens new window) zu installieren, Binrdateien von unserer Release-Seite (opens new window) herunterzuladen oder es selbst zu kompilieren.

# Binrdatei

Von der Release-Seite (opens new window) kann Nushell bereits kompiliert heruntergeladen werden. Alternativ kann Nushell, wenn Homebrew (opens new window) verwendet wird, mit dem Befehl brew install nushell installiert werden. Unter Windows knnen Winget (opens new window) oder Chocolatey (opens new window) zur Installation verwendet werden: winget install nushell beziehungsweise choco install nushell.

# Windows

Achtung: Nu funtioniert aktuell auf Windows 10 und hat keine Untersttzung fr Windows 7/8.1.

Die aktuelle, verffentlichte .zip-Datei von der Release-Seite (opens new window) herunterladen und den Inhalt extrahieren nach:

C:\Program Files

Danach den nu beinhaltenden Ordner der Umgebungsvariable PATH hinzufgen. Wenn das passiert ist, kann nu wie folgt gestartet werden:

> nu
C:\Users\user>

Fr Nutzer des Windows Terminal (opens new window) kann nu als Standard-Shell gesetzt werden, indem:

{
  "guid": "{2b372ca1-1ee2-403d-a839-6d63077ad871}",
  "hidden": false,
  "name": "Nu Shell",
  "commandline": "nu.exe"
}

zu "profiles" in den Terminal Einstellungen (JSON-Datei) hinzufgt wird. Zu guter Letzt, muss nur noch "defaultProfile" angepasst werden:

"defaultProfile": "{2b372ca1-1ee2-403d-a839-6d63077ad871}",

Jetzt sollte sich nu beim Start von Windows Terminal ffnen.

# Vorbereitungen

Bevor Nu installiert werden kann, muss sichergestellt werden, dass das System die ntigen Anforderungen erfllt. Aktuell bedeutet das sicherzustellen, dass die Rust-Toolchain und lokale Abhngigkeiten installiert sind.

# Installieren einer Compiler-Suite

Damit Rust richtig funktioniert, muss eine kompatible Compiler-Suite auf dem System installiert sein. Die empfohlenen Compiler-Suites sind:

  • Linux: GCC oder Clang
  • macOS: Clang (Xcode installieren)
  • Windows: MSVC (installieren von Visual Studio Community Edition (opens new window) oder der Visual Studio Build Tools (opens new window))
    • Sicherstellen, dass "Desktop development with C++" installiert wird
    • Jede Visual Studio Edition sollte funktionieren (Community ist gratis)

# Rust installieren

Wenn Rust noch nicht auf dem System installiert ist, ist rustup (opens new window) der beste Weg es zu bekommen. Rustup ist ein Programm um Rust-Installationen zu managen wie beispielsweise verschiedene Rust-Versionen.

Nu bentigt aktuell die latest stable (1.55 oder neuer) Version von Rust. Der einfachste Weg ist es, rustup die korrekte Version fr finden zu lassen. Wenn rustup zum ersten Mal gestartet wird, wird nachgefragt, welche Version installiert werden soll:

Current installation options:

default host triple: x86_64-unknown-linux-gnu
default toolchain: stable
profile: default
modify PATH variable: yes

1) Proceed with installation (default)
2) Customize installation
3) Cancel installation

Wenn Sie bereit sind, drcken Sie 1 und dann Enter.

Wenn Rust nicht via rustup installiert werden soll, knnen auch andere Methoden verwendet werden um es zu installieren (z.B. vom Paketmanager der Linux-Distro). Dabei muss sichergestellt werden, dass es sich um Rust-Version 1.55 oder neuer handelt.

# Abhngigkeiten

Es mssen die Pakete pkg-config und libssl-dev installiert werden:

Linux-Nutzer die die optionalen Funktionen rawkey und clipboard verwenden mchten, mssen auerdem die Pakete libx11-dev und libxcb-composite0-dev installieren:

# macOS

Mittels Homebrew (opens new window), mssen die Pakete openssl und cmake ber folgenden Befehl installiert werden:

# Installieren von crates.io (opens new window)

Wenn alle die Abhngigkeitenn, die fr Nu bentigt werden, installiert sind, kann cargo verwendet werden um Nu zu installieren.

> cargo install nu

Das war's! Cargo wird Nu und seine anderen Abhngigkeiten herunterladen, kompilieren und schlielich im cargo bin Pfad installieren, damit es benutzt werden kann.

Wenn mehr Funktionalitten installiert werden sollen, kann der folgende Befehl verwendet werden:

> cargo install nu --features=extra

Um alle verfgbaren Funktionalitten zu bekommen, ist es am einfachsten einen Checkout durchzufhren und es selbst mit Hilfe der Rust-Tools zu kompilieren:

> git clone https://github.com/nushell/nushell.git
> cd nushell
nushell> cargo install --path . --features=extra

Damit das funktioniert, sollte sichergestellt werden, dass alle oben genannten Abhngigkeiten auf dem System installiert sind.

Wenn Nu schlielich installiert ist, kann die Shell mit dem nu-Befehl gestartet werden:

$ nu
/home/jt/Source>

# Kompilieren von Quelldateien

Nu kann auch direkt aus den Quelldateien, die auf GitHub verfgbar sind, kompiliert werden. Das stellt unmittelbar die neuesten Funktionen und Fehlerbehebungen von Nu zur Verfgung.

> git clone https://github.com/nushell/nushell.git

Git clont das main nushell Repo. Von da aus, kann Nu, wenn rustup verwendet wird, wie folgt kompiliert und gestartet werden:

> cd nushell
nushell> cargo build --workspace --features=extra && cargo run --features=extra

Nu kann auch in "release" Modus kompiliert und gestartet werden:

nushell> cargo build --release --workspace --features=extra && cargo run --release --features=extra

Leute, die sich mit Rust auskennen, wundern sich womglich, warum hier sowohl ein build als auch ein run durchgefhrt wird, obwohl run standardmig auch einen Build durchfhrt. Das ist ntig, um ein Problem mit der neuen default-run-Option von Cargo zu umgehen, damit alle Plugins kompiliert werden. Dies wird unter Umstnden in Zukunft nicht mehr ntig sein.

# Als Login-Shell verwenden

!!! Nu befindet sich noch in der Entwicklung und ist unter Umstnden nicht stabil genug fr die tgliche Nutzung. !!!

Um die Login-Shell festzulegen, kann der Befehl chsh (opens new window) verwendet werden. Manche Linux-Distributionen haben eine Liste von erlaubten Shells in /etc/shells und verbieten es die Shell zu ndern, bis Nu in der Whitelist ist. Wenn die shells-Datei nicht abgendert wurde, erscheint vielleicht einen hnlichen Fehler, wie:

chsh: /home/username/.cargo/bin/nu is an invalid shell

Nu kann zur Liste der erlaubte Shells hinzugefgt werden, indem der Pfad von nu der shells-Datei angefgt wird. Der Pfad, der hinzugefgt werden muss, kann mit dem Befehl which nu herausgefunden werden. Normalerweise ist es $HOME/.cargo/bin/nu.

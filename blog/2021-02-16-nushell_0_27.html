# Nushell 0.27

Nushell, or Nu for short, is a new shell that takes a modern, structured approach to your commandline. It works seamlessly with the data from your filesystem, operating system, and a growing number of file formats to make it easy to build powerful commandline pipelines.

Today, we're releasing 0.27 of Nu. This release fixes some long-standing issues with Nushell.

# Where to get it

Nu 0.27 is available as pre-built binaries (opens new window) or from crates.io (opens new window). If you have Rust installed you can install it using cargo install nu.

If you want all the goodies, you can install cargo install nu --features=extra.

As part of this release, we also publish a set of plugins you can install and use with Nu. To install, use cargo install nu_plugin_<plugin name>.

# Improvements

# New commands (fdncred)

  • A new term size commands lets you get the current size of the terminal (opens new window)
  • New pow operator (opens new window)
  • New modulus(%) operator (opens new window)

# Functionality (jonathandturner, WatsonThink, ilius, andrasio, Qwanve, fdncred, ammkrn)

  • Tables now truncate instead of crashing when too large (opens new window)
  • source command now can use tilde in the path (opens new window)
  • let-env can now shadow environment variables (opens new window) in addition to adding new ones
  • Prompts can now call external commands on initial startup (opens new window)
  • You can now configure the file size standard to use (opens new window)
  • Strings can now be passed in where column-paths were expected (opens new window) and here (opens new window)
  • More file size types have been added (opens new window)
  • sort-by can now reverse sort (opens new window)
  • which can now take multiple applications (opens new window)
  • ps -l now has a cleaner output when the parent is missing (opens new window)
  • select can no longer select the same column twice (opens new window)
  • to-md --per-element now gives you more fine-grained control over the markdown output (opens new window)
  • A new $nothing built-in variable to help check for the existence of a value (opens new window)
  • str from can convert more things to strings (opens new window)
  • Booleans are now shown as true/false instead of Yes/No (opens new window)
